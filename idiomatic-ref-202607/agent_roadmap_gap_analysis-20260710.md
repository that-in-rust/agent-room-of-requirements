# Agent Roadmap Gap Analysis

The seed title names roadmap gap analysis but provides no opening definition of a gap, decision owner, prioritization horizon, or evidence threshold. This section helps decide whether a recurring unmet workflow need justifies creating, extending, consolidating, or declining an agent capability.

**Recommended default.** Treat every proposed agent as an evidence hypothesis: document repeated demand, compare existing capabilities, validate a narrow intervention, and promote it only after measurable reuse. New agents add discovery, maintenance, trigger-overlap, and verification costs, so observed work must justify a durable abstraction rather than a one-off prompt.

**Operating conditions.** This default works when several comparable tasks expose the same missing decision support, ownership is clear, and a reversible pilot can test value.

**Failure boundary and recovery.** It fails when demand is anecdotal, an existing skill already covers the job, the problem is project-specific, or the proposal begins with a preferred tool rather than a user gap. The bounded recovery is: Alternatives include improving an existing skill, adding a reference, writing a reusable job, changing project instructions, or deliberately leaving the workflow manual. Do not force a roadmap decision from missing evidence.

**Gotchas and counterexample.** Key risks are solution-first roadmaps, duplicate agents, vague triggers, maintenance without owners, and prioritizing novelty over avoided failure. Good: three repeated incident reviews reveal the same evidence-triage gap and a narrow pilot reduces rework. Bad: create an agent from one interesting prompt. Borderline: add a reference instead of an agent when reasoning is reusable but autonomy is unnecessary.

**Verification and confidence.** Verify demand records, overlap analysis, pilot outcomes, trigger tests, maintenance ownership, and explicit kill criteria. It is strongly defensible that roadmap entries need demand, scope, ownership, and verification evidence. The minimum recurrence count and value threshold are portfolio judgments, not universal constants.

**Assumptions and second-order consequence.** Assume a repository or team can observe repeated workflows and can test a candidate without irreversible rollout. A roadmap gap is a missing reliable outcome, not an empty slot in an agent taxonomy.

**Revision outcome.** Add an opening gap-definition contract, evidence ladder, exclusion rules, and promotion or kill outcomes.
