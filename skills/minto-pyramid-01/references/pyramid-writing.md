Source basis:
- Barbara Minto, *The Minto Pyramid Principle: Logic in Writing, Thinking, and Problem Solving*
- Barbara Minto, *The Minto Pyramid Principle Self-Study Course Workbook*

## Core rules

Use these as non-negotiables:

1. Make the whole document drive toward one governing thought.
2. Present from the top down.
3. Make each parent summarize the ideas below it.
4. Keep sibling ideas the same kind of idea.
5. Put sibling ideas in a clear logical order.
6. Make each child group answer the question raised by the point above it.

If any one of these breaks, the structure is probably wrong.

## Top-down build

Use this sequence when the subject is at least partly clear:

1. State the subject.
2. State the reader's question.
3. State the answer.
4. State the Situation.
5. State the Complication.
6. Recheck that the Complication really triggers the Question.
7. Find the new question raised by the Answer.
8. Build the Key Line.
9. Repeat the same question-answer process downward.

Use the introduction to prove that you are answering the right question, not to provide generic background.

## SCQA introduction

Use this shape:

- Situation: what the reader already knows or will accept
- Complication: what changed, failed, appeared, or became urgent
- Question: what the reader should now be asking
- Answer: what the document says

Treat the Complication as a trigger in the story, not merely as a complaint.

## Key Line rules

The Key Line answers the new question raised by the Answer.

- If the Answer says "we should do X," the Key Line may be reasons or changes.
- If the Answer says "we can achieve Y," the Key Line may be steps.
- If the Answer says "option A is best," the Key Line may be criteria or advantages.

Put ideas in the Key Line, not topics.

Bad Key Line labels:

- background
- findings
- recommendations
- issues

Good Key Line labels are actual statements.

## Deduction and induction

### Deduction

Use deduction when proving a conclusion.

Checks:

- Make the second point comment on the first.
- Make the conclusion follow from the chain.
- Keep the chain short and readable.

### Induction

Use induction when multiple similar ideas imply a higher insight.

Checks:

- Name the grouping with a plural noun.
- Remove misfits.
- Do not infer more than the evidence warrants.

Useful plural nouns:

- reasons
- problems
- steps
- changes
- opportunities
- risks
- conclusions

If you cannot name the class, the grouping is probably invalid.

## Grouping and ordering

Group only like-with-like:

- reasons with reasons
- steps with steps
- problems with problems
- changes with changes

Order each group by one visible logic:

- deductive
- time
- structural
- degree or priority

If the order feels arbitrary, the thinking is unfinished.

## End-product wording

Rewrite vague action ideas into visible end products.

Weak:

- improve reporting
- strengthen coordination
- enhance efficiency

Better:

- install a reporting system that flags change early
- assign planning responsibility to the regions
- establish a follow-up system for overdue accounts

Ask:

- What will exist when this step is complete?
- What will someone literally have in hand?
- How will we know the step is done?

## Distinguish levels of action

Do not mix:

- a result
- the actions needed to produce that result

Keep a result above the steps that generate it.

If sibling points include both a result and a substep, split the levels.

## Headings and page structure

Use headings to reveal the structure, not to replace it.

- Reflect the main point in the title.
- Use headings that state ideas.
- Keep parallel points in parallel grammatical form.
- Keep headings short.
- Introduce each heading group before the headings appear.
- Make the prose read smoothly even if the headings disappeared.

Use numbering, indentation, or dot-dash outlines only to make the hierarchy visible.

## Transitions and summaries

Introduce each major group by:

- telling a brief story that leads to it
- or referencing backward to the prior point

Summarize long sections before moving on.

If you include a conclusion:

- add significance
- add perspective
- or set up the next action

Avoid weak endings that merely repeat what the reader already knows.
