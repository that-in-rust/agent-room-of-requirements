# Algorithmic Philosophy Template

Use this when the user wants the art system explained as an intentional movement rather than a pile of implementation notes.

## Template

### Movement Name

Pick one short name with texture.
Examples:
- `Organic Turbulence`
- `Quiet Interference`
- `Luminous Drift`

### Concept Seed

State the emotional or mathematical premise in one sentence.

Examples:
- "This piece treats memory like particles losing cohesion in a slow current."
- "This piece treats traffic as a pressure field that folds light into lanes."

### Core Principles

Write 3 to 5 bullets:
- what moves
- what constrains it
- what introduces variation
- what makes each run unique
- what the viewer should feel

### Translation To Code

Convert the philosophy into implementation choices:

| philosophy idea | code move |
| --- | --- |
| gentle drift | low-frequency noise field |
| memory decay | alpha fade plus sparse respawn |
| tension release | spring easing after threshold |
| hidden rhythm | phased oscillators or periodic emitters |

### Parameter Surface

Keep it compact:
- `seed`
- `palette`
- `density`
- `tempo`
- `noise_scale`
- `trail_decay`

### Failure Check

If the philosophy can only be felt after a paragraph of explanation, the system is too abstract or too weak. Strengthen the behavior, not the prose.
