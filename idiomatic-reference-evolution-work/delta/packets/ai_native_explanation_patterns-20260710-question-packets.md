## Section 026: Evidence Boundary Notes

### Question 01: What decision does this reference help make?

- **counterargument_or_limit:** Too many inline labels can interrupt a beginner's teaching flow.
- **assumptions_and_boundaries:** Keep detailed classification in the working record and surface labels wherever origin or confidence changes action.
- **revision_decision:** Preserve the three labels and add a claim-status taxonomy plus presentation rules.
- **additional_insight_to_add:** Evidence status is part of the concept being taught; hiding it for simplicity can teach unwarranted certainty.

### Question 02: What is the recommended default, and why?
- **current_section_observation:** "Tied to a path or URL" is insufficient because citation presence does not show whether the source was inspected, applicable, primary, or current.
- **supporting_reason:** Classify each material claim by origin, inspection status, scope, evidence type, confidence, and transformation before drafting.
- **counterargument_or_limit:** Connective and definitional prose may not need per-sentence provenance.
- **assumptions_and_boundaries:** Apply claim-level detail to causal, quantitative, historical, normative, and action-bearing statements.
- **revision_decision:** Add a materiality rule and a structured claim record.
- **additional_insight_to_add:** One sentence may contain several claims with different evidence status and should be split when a single label would mislead.

### Question 03: When does the default work well?
